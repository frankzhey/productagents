# SPKChallenge Part 2 Mini Program UX 方案

## 1. 页面地图
- 页面列表
  - Main 口语测评首页
  - Recording Page 开始测评页
  - Waiting for Score 测评状态页
  - Score 测评结果页
  - Activity Ended 活动结束态
- 页面之间关系
  - 首页 -> 开始测评页：用户未提交且活动进行中时可进入
  - 首页 -> 等待页：用户已提交但尚未出分时进入
  - 首页 -> 结果页：用户已出分时进入
  - 首页 -> 活动结束态：活动已结束且用户未提交时展示
  - 开始测评页 -> 等待页：用户上传录音成功后进入
  - 等待页 -> 结果页：系统已收到结果回传后进入
  - 已提交用户在活动结束后仍保留首页 -> 等待页 / 结果页路径

## 2. 用户流程
- 用户路径
  - 进入小程序活动首页
  - 微信授权并完成身份识别
  - 系统判断当前参与状态和活动状态
  - 未提交用户进入录音页完成录音与上传
  - 已提交未出分用户查看等待文案并可分享
  - 已出分用户进入结果页查看 CEFR 等级与固定结果文案
- 操作步骤
  - Step 1：用户打开活动首页并阅读活动说明
  - Step 2：用户点击“立即参与”或“开始口语测评”后触发微信授权
  - Step 3：授权成功后系统识别 `unionId` 并校验是否已参与
  - Step 4：未参与用户进入录音页，系统随机展示 38 套题目中的 1 套题图
  - Step 5：用户点击录音按钮开始 2 分钟倒计时录音
  - Step 6：用户可手动结束录音，也可等待系统自动结束
  - Step 7：录音结束后用户可重新录音、播放或上传录音
  - Step 8：上传成功后用户进入等待页
  - Step 9：结果回传后用户从首页或等待态进入结果页
- 页面跳转
  - 首页 CTA 与状态强绑定，避免用户误入错误流程
  - 活动结束时仅未提交用户停留在结束态，已提交用户仍可查看等待页或结果页
  - 结果页鼓励文案在 `submissionId` 首次生成时随机一次，后续进入保持不变

## 3. 页面结构
### Main 口语测评首页
- Header
  - British Council 风格品牌识别区
  - 活动状态标签
  - 活动规则快捷入口
- Content
  - 主 Banner 区
  - 活动主标题与价值说明
  - 参与规则卡片
  - 流程说明卡片
  - 用户当前状态卡片
- Action
  - 立即参与按钮
  - 开始口语测评按钮
- 状态变体
  - 未提交：两个按钮均可点击
  - 已提交未出分：两个按钮均展示“等待测评结果”且禁用
  - 已出分：两个按钮均展示“查看结果”且可点击
  - 活动结束未提交：两个按钮均展示“谢谢参与，活动结束”且禁用，Banner 切换为结束素材

### Recording Page 开始测评页
- Header
  - 返回按钮
  - 页面标题
  - 录音规则说明
- Content
  - 题目图片区
  - 题目标题与提示区
  - 录音状态卡片
  - 倒计时显示区
  - 录音波形占位区
- Action
  - 开始录音按钮
  - 结束录音按钮
  - 重新录音按钮
  - 播放按钮
  - 上传录音按钮
- 关键状态
  - 默认态：题图已加载，等待录音
  - 录音中：倒计时走动，主按钮切换为结束录音
  - 录音完成待确认：展示重录、播放、上传三按钮
  - 上传中：上传按钮 loading，不允许重复点击
  - 上传失败：保留当前录音并展示重试提示

### Waiting for Score 测评状态页
- Header
  - 页面标题
  - 状态进度徽标
- Content
  - 状态插画区
  - 固定文案区
  - 预计等待时长说明区
  - 分享收益提示区
- Action
  - 分享给好友按钮
  - 分享到朋友圈按钮
  - 返回首页按钮

### Score 测评结果页
- Header
  - 页面标题
  - 结果状态标签
- Content
  - CEFR 等级主视觉图区
  - 等级背景图层
  - 等级名称与一句话标签
  - 鼓励文案区
  - IELTS and the CEFR 链接区
  - 结果说明卡片
- Action
  - 返回首页按钮
  - 再次分享按钮
- 结果内容规则
  - CEFR 支持 `A1`、`A2`、`B1`、`B2`、`C1`、`C2`
  - 每个等级有独立背景图与结果图
  - 每个等级对应 5 条文案池
  - 对同一 `submissionId` 首次随机选中 1 条后固定显示

## 4. 核心组件
- Hero Banner 组件
  - 支持活动进行中与活动结束两套素材
  - 支持状态标签、主标题、副说明、浮层提示
- 双 CTA 组件
  - 两个按钮共享同一状态源
  - 包含默认、hover、active、disabled 四种状态
- 状态卡片组件
  - 展示 `activityStatus`、`submissionStatus` 和当前下一步提示
- 题目图片卡组件
  - 承载随机题图和题目引导说明
- 录音控制组件
  - 支持开始、结束、重录、播放、上传五类行为
  - 包含倒计时、波形占位、状态标签和文件存在校验
- 确认弹窗组件
  - 用于重新录音确认
  - 明确覆盖 Confirm、Cancel、关闭三种退出方式
- Toast / Inline Feedback 组件
  - 用于授权失败、录音失败、上传成功、上传失败、分享反馈
- CEFR 结果卡组件
  - 显示等级图、对应背景、固定结果文案、学习链接

## 5. 交互说明
- 筛选 / 搜索
  - 本期前台页面无复杂筛选和搜索
- 弹窗
  - 用户点击“重新录音”时弹出确认提示：之前的录音内容将被删除
  - 用户点击 Confirm：删除旧录音并回到录音中状态
  - 用户点击 Cancel 或关闭：保留当前录音并停留在确认后处理态
- 校验
  - 未授权不得进入录音流程
  - 已提交用户不得再次进入录音提交流程
  - 录音文件不存在时不得触发上传
  - 上传中不得重复点击上传按钮
- 提交
  - 上传录音时按钮进入 loading 状态，避免重复点击
  - 上传成功后立即切换到等待态
- 提示反馈
  - 授权失败：显示“授权失败，请稍后重试”
  - 上传成功：显示“上传成功，等待测评结果”
  - 上传失败：显示“提交失败，请稍后重试”
  - 分享触发：显示已调起微信分享能力
  - 活动结束：显示“谢谢参与，活动结束”

## 6. 状态与数据需求
- 默认 / loading / 空 / 错误 / 成功
  - 首页
    - 默认：展示活动 Banner、规则、CTA
    - loading：加载用户状态与活动配置
    - 错误：状态获取失败时展示重试入口
    - 成功：按用户状态渲染 CTA
  - 录音页
    - 默认：题图已加载，等待录音
    - loading：初始化权限和题图资源
    - 错误：权限被拒绝、题图缺失、上传失败
    - 成功：录音草稿可播放并可上传
  - 等待页
    - 默认：展示固定文案和分享入口
    - loading：查询结果状态
    - 成功：仍待评分时保持等待内容
  - 结果页
    - 默认：展示等级图、背景图、固定文案
    - 错误：结果字段异常时展示兜底卡片
- 数据字段
  - `unionId`
  - `activityStatus`
  - `submissionStatus`
  - `campaignId`
  - `questionId`
  - `questionImageUrl`
  - `recordingDuration`
  - `audioFileId`
  - `audioFileUrl`
  - `submissionId`
  - `cefrLevel`
  - `cefrImageUrl`
  - `cefrBackgroundUrl`
  - `resultCopy`
  - `sharePath`
- API 依赖
  - 获取用户状态接口
  - 获取活动配置接口
  - 获取随机题图接口
  - 上传录音接口
  - 查询结果接口
  - 结果素材映射接口或静态资源配置

## 7. 响应式说明
- Desktop
  - 用于评审展示，采用左侧说明板 + 右侧多手机框预览结构
  - 支持在一个页面中切换首页、录音页、等待页、结果页和结束态
- Tablet
  - 收敛为上下结构，说明板置顶，手机预览区置底
  - 保留所有关键交互控件
- Mobile
  - 以单列纵向布局为主
  - CTA、录音按钮与分享按钮均使用大点击区
  - 结果页优先展示 CEFR 主视觉，学习链接与分享操作置于次级区块
