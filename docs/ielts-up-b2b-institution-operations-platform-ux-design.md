# IELTS Up B2B Institution Operations Platform UX Design

## 1. 页面地图

### 页面列表

- 登录页：账号登录、机构识别、首次登录改密入口
- 首页工作台：欢迎区、配额卡片、智能提醒、经营指标、快捷操作
- 组织与权限：组织结构、成员管理、用户管理、角色管理、修改密码
- 客户与订阅：客户列表、客户详情、合同管理、订阅详情、权益查看、额度总览、使用历史
- 内容与试卷：题库列表、题目详情、试卷列表、创建试卷、试卷详情
- PIN 码中心：PIN 列表、创建 PIN、PIN 详情、分发记录
- 学员与教学执行：学员列表、学员详情、班级管理、分组管理、Arrange TDP、Manage TT、Upload TT
- 报告中心：结果总览、产品线报告列表、报告详情
- API 管理：API 配置、调用记录、异常筛查

### 页面之间关系

- 登录页进入首页工作台。
- 首页工作台通过快捷入口跳转到组织、订阅、PIN、试卷、学员、报告、API 页面。
- 客户详情与合同、订阅、权益、额度、使用历史互相串联，构成商业链路。
- 题库与试卷页面服务于学员和教学执行页面。
- 学员与教学执行页面生成的结果流向报告中心。
- API 管理与调用记录形成开放能力闭环。

## 2. 用户流程

### 用户路径

- 平台管理员：登录 -> 首页查看异常与到期提醒 -> 管理客户/合同/订阅 -> 分配权益与 PIN -> 查看 API 接入状态
- 机构管理员：登录 -> 首页查看额度与待办 -> 配置组织与成员 -> 管理学员与班级 -> 查看报告与用量
- 教学用户：登录 -> 首页快捷进入试卷/学员 -> 选题组卷 -> 安排训练 -> 上传 TT -> 查看报告详情
- 销售运营：登录 -> 客户列表 -> 合同与订阅管理 -> 跟踪额度使用 -> 辅助续费与扩容

### 操作步骤

1. 用户登录并进入角色化首页。
2. 系统根据角色展示额度、提醒和快捷操作。
3. 用户进入具体业务模块完成管理动作。
4. 系统根据业务结果更新额度、状态和报告。
5. 用户回到首页或报告中心复盘。

### 页面跳转

- 首页配额卡片 -> 额度总览 / 权益详情
- 首页提醒 -> 合同详情 / 使用历史 / API 调用记录 / 报告详情
- 题库列表 -> 题目详情 -> 创建试卷 -> 试卷详情
- 学员列表 -> 班级管理 / Arrange TDP / Upload TT
- 报告中心 -> 产品线报告详情
- API 配置 -> API 调用记录

## 3. 页面结构

### 全局框架

- Header：品牌标识、全局搜索、机构切换、通知入口、用户菜单
- Sidebar：一级导航按业务域分组，支持折叠
- Content：当前模块主视图，采用卡片与表格混合布局
- Action：右上角主操作按钮、筛选器、导出、批量操作

### 首页工作台

- Header 区：欢迎文案、当前机构、时间范围切换
- Summary 区：产品配额卡片、到期状态、使用进度
- Alert 区：智能提醒列表，支持一键跳转处理
- Insight 区：指标标签页，展示机构运营概览
- Quick Action 区：创建试卷、生成 PIN、导入学员、查看报告

### 列表型页面

- 顶部：标题、筛选、搜索、主操作按钮
- 主体：表格或列表卡片
- 右侧或弹窗：详情预览、编辑表单、二次确认

### 详情型页面

- 顶部：返回、标题、状态标签、主次操作
- 主体左侧：核心信息、状态流、关联对象
- 主体右侧：日志、使用记录、提醒信息

## 4. 核心组件

- 配额卡片：展示产品线、剩余额度、已用额度、到期时间、状态色
- 智能提醒卡：展示提醒类型、优先级、处理建议、跳转按钮
- 经营指标标签页：按产品线或时间切换指标视图
- 业务表格：客户、成员、学员、合同、PIN、API 调用记录统一表格组件
- 状态标签：用于展示草稿、生效、到期、异常、处理中、完成
- 侧滑详情面板：快速查看详情，减少页面跳转
- 创建弹窗：创建试卷、创建 PIN、添加成员等轻量操作
- 筛选栏：产品线、状态、机构、时间、角色、多字段搜索
- 上传组件：支持 Upload TT 的拖拽上传、格式提示、进度、失败原因
- 报告摘要卡：展示总分、趋势、最近更新时间、进入详情入口

## 5. 交互说明

- 筛选：列表页统一采用顶部筛选栏，筛选条件变更后即时刷新列表。
- 搜索：支持按名称、编号、邮箱、机构名、产品线关键词搜索。
- 弹窗：轻量新增和确认动作使用弹窗，复杂编辑使用侧滑面板或独立详情页。
- 校验：表单字段必须有 label、必填标识、错误提示；错误提示紧贴字段显示。
- 提交：主按钮仅保留一个；提交中显示 loading 态并禁用重复点击。
- 提示反馈：成功操作采用顶部 toast 或卡片内反馈，失败操作需给出原因与建议动作。
- 快捷操作：首页快捷入口按角色动态显示，避免无权限入口干扰。
- 表格操作：高频动作保持在行内，低频动作收纳到更多菜单。
- 上传交互：上传前显示格式限制，上传中显示进度，失败后支持重传。
- 报告查看：支持列表筛选、详情跳转和状态提示；当报告生成中时展示占位信息。

## 6. 状态与数据需求

### 默认状态

- 首页展示最近一次可用数据快照。
- 列表页展示默认筛选结果和总数。
- 表单页加载推荐默认值与帮助说明。

### Loading

- 首页卡片与图表采用骨架屏。
- 表格区域采用行骨架屏或局部 spinner。
- 上传与异步任务使用进度条和阶段文案。

### 空状态

- 无客户、无学员、无试卷、无 API 记录时，展示说明文案和主操作入口。
- 报告中心无数据时，提示先完成训练或筛选调整。

### 错误状态

- 接口失败时保留页面结构，仅替换局部内容为错误卡。
- 异步处理失败时展示错误原因、重试入口和帮助说明。
- 无权限访问时展示受限提示，而不是空白页。

### 成功状态

- 创建、编辑、分配、上传完成后有显式成功反馈。
- 涉及状态变更的页面需同步更新状态标签和时间戳。

### 数据字段

- 首页：机构名称、产品线、剩余额度、已用额度、到期时间、提醒级别、最近操作
- 组织：部门名称、成员姓名、账号、角色、状态、最后登录时间
- 客户合同：客户名称、合同编号、产品线、订阅周期、状态、生效时间、到期时间
- PIN：批次号、产品线、数量、状态、有效期、分发对象
- 学员：学员姓名、学号、班级、分组、训练状态、最近报告
- 报告：产品线、学员、总分、维度分、生成时间、状态
- API：应用名称、Key 标识、权限范围、最近调用时间、错误码、状态

### API 依赖

- 组织权限服务
- 客户/合同/订阅服务
- 权益/额度/使用历史服务
- 题库与试卷服务
- 学员与训练任务服务
- 报告聚合服务
- API 管理与调用审计服务

## 7. 响应式说明

### Desktop

- 采用左侧固定导航 + 顶部工具栏 + 多列内容区布局。
- 首页核心信息优先两到三列卡片展示。
- 表格页支持完整筛选和行内操作。

### Tablet

- 侧边导航折叠为图标栏或抽屉。
- 首页卡片改为双列，详情侧栏下移为底部面板。
- 表格保留关键列，次要信息折叠到详情。

### Mobile

- 导航收纳为抽屉。
- 首页卡片改为单列堆叠。
- 表格改为列表卡片，行操作收纳为底部操作菜单。
- 高风险操作保持单独确认，确保点击区域不小于 40x40。
