# Decision Checklist - spkchallengpart2-miniprogram

## 使用方式

本清单用于 PM / UX / Engineering Review 在评审前快速拍板关键业务决策。

建议处理方式：

1. 先确认 P0 项，避免影响 Story 边界和 AC 准确性。
2. 对未确认项，可先接受“推荐默认项”进入下一阶段，再在评审中微调。
3. 一旦确认，请同步回写 PRD 的 `Open Questions` 与 `Rules/SPKChallenge-rules.md`。

## 建议默认结论

如需快速推进到 UX / Eng Review，建议先采用以下默认结论：

1. 接受 D1：唯一成功提交按 `unionId + campaignId` 约束。
2. 接受 D2：活动结束不影响已提交用户查看等待页或结果页。
3. 接受 D3：重录不重抽题，同一录音尝试内保持 `questionId` 不变。
4. 接受 D4：结果鼓励文案在首次生成后固定到 `submissionId`。
5. 接受 D5-D7：分享回到活动首页，并保留来源参数与分类埋点。
6. 接受 D8-D10：回调默认 3 次重试 + 人工补偿，资源缺失走通用兜底页。
